Count — how many rows match, without loading them

db.count(entity, opts) returns just the number of matching rows. Use it for pagination totals, badge counts, and quick checks where you don't need the data itself. Same filter vocabulary as list.

How it works

  • db.count(entity, opts) returns a Promise<number>. opts accepts the same filters as list — sort, pagination, and projection are ignored (they can't change the count).
  • db.countSync(entity, opts) is the synchronous version for the in-memory backend.
  • For indexed filter fields, count walks the index. Otherwise it walks the data and counts matches — same cost as list(...).length but without allocating an array.
  • When to use: "how many unread messages", "records on this page", pagination total, boolean "does anything match".
  • The Count vs list verification panel below shows that count(opts) === list(opts).length always.

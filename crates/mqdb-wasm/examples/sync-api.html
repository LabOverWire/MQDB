Sync API — no await, direct returns

The in-memory backend supports a synchronous mirror of every CRUD method: createSync, readSync, updateSync, deleteSync, listSync, countSync. No Promises, no microtasks — useful for tight loops, tests, and synchronous event handlers where you can't await.

Checking backend…

How it works

  • The async API (create, list, …) always returns a Promise, so it works on any backend.
  • The sync API (createSync, listSync, …) returns the value directly and only works when the data lives in memory — for IndexedDB and encrypted backends it throws.
  • Check at runtime with db.isMemoryBackend(); true means the sync methods are safe to call.
  • When to use sync: tight loops (seeding test data), synchronous hot paths, code that already owns a Database and doesn't want to propagate async. The sync path avoids one microtask hop per call.
  • When to use async: anything that might later run against IndexedDB or encrypted storage; any code path that already uses await anyway — the sync/async perf gap disappears once you have I/O.
